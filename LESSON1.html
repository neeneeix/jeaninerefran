XHTML vs. HTML

"Stricter and cleaner version of HTML"
Aimed to replace HTML

All new web browsers support XHTML because there was a point where a huge amount of web pages had been written in the wrong syntax. Since web browsers still forced to interpret HTML the way they could, sometimes the wrong syntax could still display a logically correct output
So why XHTML?
The current market is composed of many different browser technologies. Some on computers, mobile phones, and even some on other hand-held devices. Not all of them can successfully interpret from HTML so XHTML was developed to cater to the fast-paced growth of technology.

There's actually quite a few errors in the HTML's code. For instance, the "width" & "height" are in capitals, the "&" is not encoded as "&" and the image tag doesn't isn't self-closing!

While it still produced the same output, rest assured some browsers won't be able to work with this. When I tried to code the HTML version riddled with errors into an XHTML version, Google, the browser I'm using turns out to be forgiving enough and still showed the working site. But either way, safer than sorry to work with properly encoded XHTML so you have more reassurance that your website is accessible to more people!
